
```bash

```

## Proto Schema Guidelines

### Naming Convention
```
insuretech/{domain}/{type}/v{version}/{entity}.proto

Examples:
- insuretech/authn/entity/v1/user.proto
- insuretech/policy/services/v1/policy_service.proto
- insuretech/claims/events/v1/claim_events.proto
```

### Package Naming
```protobuf
package insuretech.authn.entity.v1;
```

### Code Generation Options
```protobuf
option go_package = "github.com/labaid/insuretech/proto/authn/entity/v1;authnv1";
option csharp_namespace = "Insuretech.Authn.Entity.V1";
```

## Code Generation

### Go
```bash
cd proto
protoc --go_out=. --go-grpc_out=. insuretech/**/*.proto
```

### C#
```bash
cd proto
protoc --csharp_out=. --grpc_out=. --plugin=protoc-gen-grpc=grpc_csharp_plugin insuretech/**/*.proto
```

### Python
```bash
cd proto
python -m grpc_tools.protoc -I. --python_out=. --grpc_python_out=. insuretech/**/*.proto
```

## Contributing

### Adding New Section
1. Create new markdown file in `sections/`
2. Follow naming convention: `NN_section_name.md`
3. Update merge script if needed
4. Regenerate document

### Adding Proto Schema
1. Create file in appropriate domain folder
2. Follow proto3 syntax
3. Add comments for all fields
4. Include code generation options
5. Test with protoc compiler

### Adding Example
1. Create markdown file in `examples/`
2. Include code snippets and diagrams
3. Reference from main sections
